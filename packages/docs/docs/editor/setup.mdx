# Editor

Twigs editor is built on top of Lexical JS.

```jsx
import { Editor } from "@sparrowengg/twigs-react";
```

### Usage

```jsx live
<Editor />
```

### Handling data

The `onChange` property can be used to get the editor state, which can be stored in a state variable. You can also pass `editorState` prop to set the initial state of the editor.

The editor is not two way binded, so even if you update the `editorState` after initial render, the editor will not update.

Another method to get / set the editor state is by using the `dataManagementRef` prop. This prop takes a ref object, which can be used to get / set the editor state.

```jsx
import { DataManagementPluginHandle } from "@sparrowengg/twigs-react"

const Component = () => {
  // DataManagementPluginHandle used for typescript
  const dataManagementRef = useRef<DataManagementPluginHandle>(null);

  return (
    <Box>
      <Editor dataManagementRef={dataManagementRef} />
      <Flex>
        <Button
          onClick={() => {
            dataManagementRef.current?.getDataAsync().then((data) => {
              console.log(data.html, data.json);
              saveHtmlData(data.html);
            });
          }}
        >
          Save
        </Button>
        <Button
          onClick={() => {
            const data = someFunctionToGetSavedHtml();

            if (data) {
              dataManagementRef.current?.insertHtml(data);
            }
          }}
        >
          Import
        </Button>
      </Flex>
    </Box>
  );
};
```

Currently `getDataAsync` returns the data in HTML and JSON FormatTool, however to set the data, you can only use `insertHtml` method. Other methods and helpers will be introduced soon.
